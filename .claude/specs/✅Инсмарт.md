Продукт **InSmart** является приоритетным и открывается по умолчанию после юридического акцепта.

Ссылка на документ - как это реализовано на текущей платформе - [https://docplatforma.yonote.ru/doc/integraciya-s-insmart-tLC92pWGfM](https://dsspace.yonote.ru/doc/insmart-integraciya-cGoVzOarmf) 

Документация самого инсмарта по интеграции <https://docs.google.com/document/d/17khf_Y-lCec4xOp_yByqwJ6RnQjEY8wixzFzoIESX5M/edit?tab=t.0#heading=h.dg3ly02zmcd7>


**Суть интеграции:** Виджет Insmart используется Партнером только как инструмент для подбора продукта и оформления заявки. Вся последующая обработка данных и начисления происходят автоматически через интеграцию по API.

**Логика процесса:**

1. **Мониторинг оплаты:** Система в реальном времени получает данные от Insmart. Дальнейшая обработка и начисление вознаграждения запускаются **только в момент фактической оплаты** страхового продукта (статус «Оплачен»).
2. **Определение получателя комиссии:**
   - Система идентифицирует Партнера, совершившего сделку через виджет.
   - **Валидация «для себя»:** Если Партнер купил страховку сам себе (ФИО клиента совпадает с ФИО Партнера), система автоматически назначает автором сделки его наставника. В этом случае комиссия и баллы уходят вышестоящему партнеру.
   - Если ФК не найден при сверке данных от InSmart/Robo, система по умолчанию подставляет ID Неизвестного консультанта
3. **Синхронизация участников и продуктов:**
   - **Клиент:** Если покупатель новый, система автоматически создает ему карточку клиента и привязывает её к Партнеру. Если после проверки по мейл и телефону клиент есть в базе - то покупка зачисляется ему.
   - **Авто-наполнение каталога:** Если продан продукт или страховая компания, которых еще нет в нашей базе, система автоматически обращается к справочникам Insmart по API, берет официальные названия/коды и создает новую запись в нашей продуктовой линейке «на лету».
   - **Создание Программы:** Для каждой новой связки «Поставщик + Продукт» система автоматически создает Программу и настраивает правила начисления комиссии.
4. **Регистрация финансового результата:**
   - **Контракт:** Система создает в реестре новый контракт в статусе «Активирован». Сумма контракта — это полная стоимость полиса для клиента. Контракт получает уникальный номер и пометку «Заказ из Insmart».
   - **Транзакция:** Сразу после создания контракта формируется транзакция на выплату. Сумма транзакции — это размер агентской комиссии, которую передает Insmart по конкретной сделке. Дальше идут начисления партнеру согласно логике Расчетов.
