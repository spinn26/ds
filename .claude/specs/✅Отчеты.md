**Назначение раздела:** Единый центр генерации, хранения и скачивания аналитических и финансовых выгрузок в формате Excel/CSV. Раздел позволяет администраторам формировать сводные и детальные данные за любой выбранный период без нагрузки на интерфейс (фоновая генерация).

## ЧАСТЬ 1. ИНТЕРФЕЙС И УПРАВЛЕНИЕ (UI)

Раздел визуально и функционально разделен на два блока: блок заказа отчета и блок архива готовых файлов.

### 1.1. Блок «Сгенерировать отчёт»

Здесь пользователь задает параметры для выгрузки данных.

- **Календарь (Период):** Два поля для выбора дат «С» и «По» (например, 01.04.2026 — 30.04.2026).
- **Тип отчёта (Выпадающий список):**
  - Реестр выплат
  - Квалификации
  - Комиссии
  - Выручка и расходы по продуктам
  - [Финрез] Транзакции
  - [Финрез] Комиссии по ФК
  - Статусы партнёров *(перенесен из другого раздела)*
- **Динамические фильтры:** Появляются в зависимости от выбранного типа отчета. Например, если выбрать «Статусы партнёров», рядом появляется дополнительный выпадающий список для выбора конкретного статуса (Терминирован, Активный, Зарегистрирован, Исключен).
- **Кнопка генерации:** Меняет свое название после выбора типа отчета (например, Сгенерировать отчёт Выручка и расходы по продуктам за период с... по...).

### 1.2. Блок «Скачать отчёты» (Архив)

Таблица, в которой сохраняется история всех заказанных ранее выгрузок. Это позволяет не генерировать один и тот же отчет дважды.

- **Фильтрация и управление:**
  - Выпадающий список Тип отчёта (для поиска нужной выгрузки в истории).
  - Кнопка Очистить фильтр.
- **Колонки таблицы:**
  - **Тип отчёта:** Название выгрузки.
  - **Статус (Цветовой индикатор):**
    - Отчёт сформирован (Светло-зеленая плашка) — файл готов к скачиванию.
    - Генерируем файл отчёта (Желтая плашка) — процесс сбора данных на сервере.
    - ошибка (красная плашка)  - процесс прервался
  - **Файл:** Синяя кнопка Скачать с иконкой облака.
  - **Период с / Период по:** За какой период собраны данные.
  - **Дата формирования:** Точные дата и время, когда администратор нажал кнопку генерации.

## ЧАСТЬ 2. БИЗНЕС-ЛОГИКА (БЭКЕНД)

1. **Асинхронная генерация:** Формирование файлов (особенно отчетов по транзакциям на десятки тысяч строк) происходит в фоновом режиме. Администратор не должен ждать загрузки страницы — он нажимает кнопку, отчет появляется в нижней таблице со статусом «Генерируем», а администратор может уйти в другой раздел.
2. **Фиксация времени:** Отчеты отражают слепок данных на момент генерации. Временные метки (Timestamp) в файлах фиксируются в стандарте ISO 8601 (UTC).

## ЧАСТЬ 3. СТРУКТУРА И СОСТАВ ОТЧЁТОВ (ДАТА-СХЕМЫ)

Ниже приведен строгий состав колонок для каждого типа выгружаемого Excel-файла.

### 1. Выручка и расходы по продуктам

**Тип:** Агрегированный (сводный) отчет для анализа прибыльности. Без привязки к конкретным датам транзакций (показывает итог за выбранный период). 

 **Колонки:**

1. Продукт (Название продукта/группы, ключ для группировки).
2. Доход (Суммарный доход по всем транзакциям продукта).
3. Расход (Суммарный расход / себестоимость).

### 2. Статус активности (Статусы партнёров)

**Тип:** Отчет по кадровым изменениям и терминации. 

**Колонки:**

1. ФИО (Партнера).
2. Статус (Например, Терминирован).
3. Фактическая дата терминации/смены статуса (Точное время фиксации решения системой, UTC).
4. Плановая дата терминации (Плановая дата отключения доступа, обычно на 1 день раньше даты терминации).

### 3. Реестр выплат

**Тип:** Финансовый срез для бухгалтерии (расчет кредиторской задолженности). 

**Колонки:**

1. ФИО, Активность (Текущий статус партнера).
2. Баланс: Сальдо, Начислено, Прочее, Пул, Итого начислено, Итого к оплате, Оплачено.
3. Реквизиты: ИП, ОГРН, ИНН, Адрес, Верифицировано (true/false).
4. Банк: Р/с, К/с, БИК, Банк.

### 4. Квалификации

**Тип:** Отчет по карьерному росту и объемам продаж партнеров. 

**Колонки:**

1. База: Партнёр, Активность.
2. Прошлый период: Предыдущая квалификация, Предыдущий ЛП, Предыдущий ГП, Предыдущий НГП.
3. Выбранный период: Текущая квалификация, Текущий ЛП, Текущий ГП, Текущий НГП.

### 5. Комиссии

**Тип:** Детальный отчет по распределению вознаграждений по каждой сделке. 

**Колонки:**

1. Сделка: Номер, Поставщик, Продукт, Программа, Клиент, Дата транзакции, Сумма транзакции.
2. Экономика компании: Доход DS, Доход без НДС.
3. Экономика партнера: ЛП, Партнёр, Комиссия, Прибыль (компании после выплат).
4. Аналитика отрыва: Комиссия до уменьшения по отрыву, Прибыль до уменьшения по отрыву (для аудита штрафов).

### 6. [Финрез] Комиссии по ФК

**Тип:** Детальный лог связи конкретного партнера, транзакции и его комиссии. 

**Колонки:**

1. Связи: Партнёр сделки, Партнёр комиссии (получатель денег), Клиент.
2. Продукт: Поставщик, Продукт, Программа.
3. Идентификаторы: Номер, ID контракта (GUID), Дата транзакции.
4. Финансы: Сумма сделки в исходной валюте, Валюта, Сумма транзакции в RUB.
5. Распределение: Доход DS до вычета НДС, Доход DS до вычета НДС RUB, Доход без НДС в RUB, Личные продажи, Комиссия, Прибыль.

### 7. [Финрез] Транзакции

**Тип:** Мастер-отчет по всем параметрам проведенных сделок. 

**Колонки:**

1. Идентификаторы: Номер, Поставщик, Продукт, Программа, Клиент, Дата транзакции.
2. Финансы: Сумма транзакции в RUB, Доход DS до вычета НДС, Доход без НДС в RUB.
3. Партнерская часть: Партнёр, Комиссия, Прибыль.
4. Специфика контракта: Сумма сделки в исходной валюте, Валюта, Тип выплаты (standard, upfront, year1 и т.д.), Кол-во оплат (факт), Срок контракта, Дата открытия контракта.
