**Назначение раздела:** Реестр для централизованного контроля подписания (акцепта) юридических, регламентных и нормативных документов партнерами платформы. Раздел носит информационный (Read-Only) характер и позволяет отслеживать, на каком этапе подписания находится каждый партнер.

## ЧАСТЬ 1. БЛОК УПРАВЛЕНИЯ И ФИЛЬТРЫ (UI)

Верхняя панель позволяет администраторам быстро находить нужных партнеров или фильтровать базу по конкретным «зависшим» документам.

**Поля фильтрации:**

- **ФИО партнера (Текстовое поле):** Поиск по полному или частичному совпадению.
- **Статус (Выпадающий список):** Выбор из значений Акцептован / Не акцептован.
- **Вид документа (Выпадающий список):** Системный фильтр, позволяющий отфильтровать реестр по конкретному типу документа из утвержденного списка (если не выбран, выводятся данные по всем документам).
- **Кнопка:** Очистить фильтры (контурная).
- **Информационная строка:** Стр. X из Y Всего записей: Z (пагинация).

## ЧАСТЬ 2. СПИСОК СИСТЕМНЫХ ДОКУМЕНТОВ

В платформу жестко зашиты 5 базовых документов, требующих акцепта от партнера:

1. Стандарты и правила работы партнерской сети ДС Консалтинг
2. Политика конфиденциальности и обработки персональных данных платформы ООО “ДС Консалтинг”
3. АГЕНТСКИЙ ДОГОВОР (ПУБЛИЧНАЯ ОФЕРТА) на привлечение клиентов
4. Согласие на фото- и видеосъемку, хранение и передачу материалов третьим лицам
5. СОГЛАСИЕ на обработку персональных данных и их передачу третьим лицам

## ЧАСТЬ 3. ОСНОВНАЯ ТАБЛИЦА (РЕЕСТР И АККОРДЕОН)

Так как у одного партнера теперь несколько документов, подписанных в разное время, таблица использует группировку (аккордеон) по профилю партнера для компактного отображения. Колонки «Источник» в новой версии нет (источник по умолчанию — Платформа).

### 3.1. Главная строка (Свернутый вид)

Отображает сводную информацию по конкретному человеку.

- **Партнёр:** Ф.И.О. (например, *Абазов Алексей Хашаович*).
- **Статус акцепта (Индикатор прогресса):** Например, 5 из 5 (все подписаны) или 3 из 5 (подписана только часть).
- **Иконка раскрытия:** Стрелочка [ v ] для просмотра детализации.

### 3.2. Развернутая строка (Детализация по клику)

При раскрытии строки партнера вниз выпадает список из 5 строк — по одной на каждый документ.

**Колонки внутреннего списка:**

- **Вид документа:** Название документа из списка выше.
- **Документы акцептованы (Чекбокс):** * Зеленый квадрат с галочкой — если документ подписан.
  - Пустой квадрат — если документ не подписан.
- **Дата акцепта:** Точная дата и время подписания (например, 19.06.2025 14:30). Если чекбокс пустой — поле даты остается пустым.

## ЧАСТЬ 4. БИЗНЕС-ЛОГИКА (БЭКЕНД)

1. **Автоматическая фиксация:** Когда партнер в своем личном кабинете нажимает кнопку «Принимаю условия / Согласен» под текстом определенного документа, система мгновенно проставляет статус «Акцептован» (зеленый чекбокс) в этом реестре и фиксирует точное время (Timestamp).
2. **Многоэтапность:** Документы могут акцептоваться не в один день (например, часть при первичной регистрации, часть — при повышении статуса до «Активен»). Реестр корректно сохраняет хронологию каждой отдельной галочки.
3. **Режим Read-only для сотрудников:** Данный раздел работает только на чтение. Сотрудник бэк-офиса не имеет технической возможности вручную поставить или снять галочку за партнера. Юридическое действие совершается только из личного кабинета самого пользователя.
4. **Версионирование (Переподписание):** Если юридический отдел обновляет текст документа (например, новую редакцию Агентского договора), бэкенд должен уметь сбросить акцепт по этому конкретному документу у всей базы (или выбранной группы). В реестре чекбокс напротив этого договора снова станет пустым, пока партнер не зайдет в кабинет и не примет новую оферту. Остальные 4 документа при этом остаются подписанными.
