
**Назначение раздела:** Ручной поиск действующих контрактов в базе, привязка к ним фактических оплат (транзакций) от клиентов, запуск алгоритма расчёта комиссионных для всей партнерской сети и фиксация результатов для последующих выплат.

### Блок 1. Поиск и выбор контракта (Верхняя таблица)

Специалист должен найти нужный контракт, чтобы привязать к нему оплату. Данные для этой таблицы подтягиваются из системной вкладки «Контракты».

**1.1. Фильтры для поиска:**

- Ф.И.О консультанта
- Ф.И.О Клиента
- Номер контракта
- Поставщик
- Продукт
- Программа

**1.2. Колонки таблицы контрактов (режим чтения):**

- Номер
- Клиент
- Партнёр
- Открыт (дата)
- Срок
- Поставщик
- Провайдер
- Продукт
- Программа
- Сумма + валюта

*Действие:* Специалист выделяет один или несколько контрактов из списка, после чего они переносятся в нижний блок в виде черновиков транзакций.

### Блок 2. Рабочая зона «Транзакции» (Нижняя таблица)

Здесь происходит непосредственный ввод финансовых данных и управление транзакциями (CRUD: Добавление, Редактирование до фиксации, Удаление).

**2.1. Элементы управления над таблицей:**

- Переключатель Показать продукт (скрывает/показывает столбец с продуктом, програмой, поставщиком).

**2.2. Колонки таблицы ввода (слева направо):**

- **Параметр** — *Выпадающий список / поле ввода*. (берется из таблицы продуктов - колонка свойство продукта - опцыонно если продукт иммет этот параметр)

### Блок 3. Панель управления и Модификаторы (Под таблицей)

**3.1. Кнопки действий:**

- **Рассчитать транзакции** — запускает математический алгоритм для всех выделенных чекбоксами строк. Во время расчёта блокируется интерфейс (появляются индикаторы загрузки). (надо упразнить эту кнопку если система позволяет упустить этот шаг и сделать предварительный расчет моментальным с момента ввода необходимых данных)  
- **Зафиксировать транзакции** — утверждает рассчитанные суммы и переносит данные в финансовые реестры.

**3.2. Системные подсказки (Чек-лист готовности):**

Под кнопкой фиксации выводится динамический список состояний. Иконки (check_box / check_box_outline_blank) меняются автоматически в зависимости от того, что заполнено в таблице выше. Это визуальные подсказки для сотрудника, подтверждающие корректность его действий.

- **Введены суммы транзакций** — галочка появляется, когда во всех добавленных в таблицу строках (черновиках) заполнено поле «Транзакция».
- **Введены даты транзакций** — галочка появляется, когда во всех строках корректно выбрана дата из календаря.
- **Расчёты не ведутся** — индикатор фонового процесса. Показывает, что математический движок в данный момент свободен (кнопка расчета не нажата и система не «висит» в ожидании).
- **Не рассчитаны комиссии по всем транзакциям** (или наоборот, *«Рассчитаны...»*) — индикатор финальной проверки. Показывает, остались ли в таблице строки с пустыми значениями в колонках «Доход DS» и «Без НДС». Если галочка не стоит, значит, сотрудник забыл нажать «Рассчитать транзакции» для части строк.

## ЧАСТЬ 2. ПРОГРАММНАЯ БИЗНЕС-ЛОГИКА (БЭКЕНД И МАТЕМАТИКА)

Описание процессов, которые происходят «под капотом» при взаимодействии специалиста с интерфейсом.

### Алгоритм расчёта (при нажатии «Рассчитать транзакции»)

Когда специалист ввёл сумму, дату и нажал кнопку расчета, система для каждой выделенной строки выполняет следующий каскад вычислений:

**Шаг 1. Конвертация в базовую валюту**

- Если Валюта = RUB, берется введенная сумма Транзакция.
- Если Валюта ≠ RUB, введенная сумма умножается на актуальный Курс валюты (см. раздел Валюты и НДС).
- *Результат: Базовая сумма в рублях.*

**Шаг 2. Расчёт чистой выручки компании (Колонка «Без НДС»)**

Система очищает сумму от издержек (эквайринг/налоги) и применяет комиссию поставщика:

- Выручка DS (Без НДС) = (Базовая сумма * % DS) / 105 * 100
- *Эта цифра выводится в интерфейсе в колонке «Без НДС» и участвует в дальнейшем распределении.*

**Шаг 3. Начисление Личного Объёма (ЛП)**

- Согласно маркетинговому плану (1 балл = 100 руб.), система переводит выручку в баллы.
- ЛП (баллы) = Выручка DS / 100

**Шаг 4. Расчёт комиссии прямого Партнёра (Колонка «Доход DS» / «Партнер»)**

- Система проверяет Квалификацию партнера, за которым закреплен контракт (Start, Pro, Expert и т.д.) и берёт его процентную ставку.
- Комиссия партнера (баллы) = ЛП * Ставка квалификации
- Комиссия партнера (рубли) = Комиссия (баллы) * 100

**Шаг 5. Фоновый расчёт структуры (Групповые продажи - ГП)**

- Система берет системный массив upperLevels (всех наставников над этим партнером).
- Проходя снизу вверх, система начисляет каждому наставнику объём **ГП** (равный рассчитанному ЛП нижестоящего).
- Система считает комиссию наставников по принципу маржинальной разницы: Выручка DS / 100 * (Процент наставника - Процент нижестоящего партнера).

  
  ***Проверка правила «Отрыва» и Проверка «Выполнения плана продаж» - должны происходить по команде от сотрудника, когда тот внесет все транзакции за период.  - можно делать в момете если разрешает система.*** 
  
  *Подробное описание расчетов и проверок правил см. Логика расчетов коммисий партнеров*
  
  *Проверка правила «Отрыва»:* Если ветка, в которой прошла транзакция, дает наставнику >70% всего его ГП, его финансовая комиссия с этой транзакции урезается на 50% (баллы объема сохраняются полностью).

  *Проверка «Выполнения плана продаж» (ОП):* Для определенных квалификаций система проверяет выполнение обязательного норматива по объему продаж за месяц. В случае невыполнения плана, к итоговой сумме комиссии со всех групповых продаж наставника применяется дополнительное удержание. При этом начисленные баллы объема и комиссия за личные продажи (ЛП) самого наставника не уменьшаются.
  

### Алгоритм фиксации (при нажатии «Зафиксировать транзакции»)

1. Система проверяет наличие расчетных данных в выделенных строках.
2. Выделенные строки удаляются из интерфейса «Транзакции» (рабочая зона очищается).
3. Транзакции со статусом «Зафиксировано» отправляются в системный раздел **«Комиссии»**.
4. Обновляются глобавые счетчики в профилях партнеров (пересчитываются ЛП, ГП, НГП текущего месяца). Если НГП партнера достигает порога перехода на новый уровень, система планирует смену его квалификации со следующего месяца.
