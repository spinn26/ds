**Назначение раздела:** Образовательный хаб платформы. Содержит базу знаний по методологии продаж и подробные курсы по каждому финансовому продукту. Успешное прохождение продуктовых курсов является системным триггером для разблокировки витрины в разделе «Продукты».

## ЧАСТЬ 1. БИЗНЕС-ЛОГИКА И ПРАВИЛА ДОСТУПА

### 1.1. Свободная навигация (Безлинейность)

- **Правило:** Все учебные блоки и курсы открыты для партнера сразу после регистрации.
- **Следствие:** Нет жесткой последовательности. Партнер может изучать продукты в любом порядке (например, начать с «Паруса», пропустив «Эволюцию»).

### 1.2. Триггер разблокировки продуктов (Связь с БД)

Это главное системное правило раздела, связывающее Обучение и Витрину продуктов.

- **Структура БД:** Каждый курс из «Блока 2» (Продукты) имеет программную привязку к конкретному Product_ID в базе данных.
- **Критерий успеха:** Курс/продукт считается изученным **только** в момент успешной сдачи итогового теста по этому курсу. Простое прокликивание видео или текста статус не меняет.
- **Системное действие:** Как только тест сдан успешно, бэкенд автоматически меняет статус этого продукта для данного пользователя с Locked на Unlocked. Продукт на витрине окрашивается в рабочие цвета и кнопка «Открыть продукт» становится активной.
- **Исключение (InSmart):** Так как InSmart открывается по умолчанию после акцепта документов, курс по нему носит исключительно информационный характер и не блокирует витрину.

## ЧАСТЬ 2. ИНТЕРФЕЙС И ФОРМАТ КОНТЕНТА (UI/UX)

Раздел представляет собой каталог с карточками курсов, разбитыми по смысловым блокам.

### 2.1. Карточка курса / Урока

На фронтенде карточка отображает:

- Название курса.
- Статус итогового теста: Не пройден, Пройден [Дата].

### 2.2. Наполнение урока (Типы контента)

Страница самого урока (плеера) поддерживает мультиформатность. Администратор может комбинировать в одном уроке:

1. **Видео-плеер** (основной фрейм окна).
2. **Аудио-дорожка** (плеер для прослушивания фоном).
3. **Текстовый блок** (лонгрид с форматированием, таблицами и графиками).
4. **Блок вложений (Кнопки):** Кликабельные кнопки для скачивания PDF-презентаций, регламентов, инструкций или перехода по внешним ссылкам.
5. **Тестирование:** Встроенный модуль проверки знаний в конце продуктового курса.

## ЧАСТЬ 4. АДМИН-ПАНЕЛЬ (УПРАВЛЕНИЕ ОБУЧЕНИЕМ)

Для сотрудников создается отдельный интерфейс управления разделом «Обучение» (LMS-модуль).

1. **Конструктор курсов:** Возможность создавать Блоки, Курсы, Уроки.
2. **Привязка к продукту:** При создании курса из Блока 2, администратор обязан выбрать из выпадающего списка связанный продукт (ID), чтобы бэкенд знал, какую карточку на витрине разблокировать после теста.
3. **Конструктор тестов:** Инструмент для создания вопросов (один правильный ответ / несколько правильных ответов).
