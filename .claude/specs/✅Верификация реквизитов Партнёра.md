**Назначение:** Автоматическая и ручная проверка юридических данных партнера для допуска к финансовым операциям, разделу «Продукты» и получению выплат.

## КОНТУР 1. АВТОМАТИЧЕСКАЯ ВЕРИФИКАЦИЯ (ИП)

Срабатывает для индивидуальных предпринимателей (ИП), данные которых идеально совпадают с государственными реестрами.

**Шаг 1. Триггер запуска** Когда новый партнер впервые пытается зайти в раздел «Продукты», система блокирует контент и выводит всплывающее окно:

- *Текст:* *Для работы с продуктами и получения вознаграждений введите ваш ИНН (ИП).*

**Шаг 2. Запрос к реестру ФНС (Бэкенд)** Партнер вводит ИНН. Система по API обращается к базе налоговой (ЕГРИП) и запрашивает выписку.

**Шаг 3. Сверка данных и автозаполнение** Система сравнивает Ф.И.О. из налоговой с Ф.И.О. в аккаунте. При полном совпадении система **автоматически заполняет** в профиле партнера следующие поля:

- Наименование ИП.
- ОГРНИП.
- Адрес регистрации (по паспорту).
- Обязательное поле для загрузки файла - Первая страница паспорта - разворот с фотографией (тут пользователь должен просто загрузить файл - по нему проверка не проводится - только на наличие файла в файла).

**Шаг 4. Переход к Шагу №2 (Ввод банковских реквизитов)**

1. Юридическим реквизитам присваивается статус **«Верифицировано»**.
2. Система выводит сообщение: *Ваши юридические данные успешно подтверждены и заполнены автоматически. Пожалуйста, проверьте их и перейдите ко второму шагу — вводу платежных реквизитов.*
3. Открывается форма для заполнения **Платёжных реквизитов**. Партнер обязан заполнить следующие поля:
   - **Наименование банка**
   - **БИК банка**
   - **Расчётный счёт**
   - **Корреспондентский счёт**
   - **Получатель** (Ф.И.О. или наименование ИП)
4. После сохранения банковских данных идет переход к следующему шагу см. Раздел Продукты доступ.

## КОНТУР 2. РУЧНАЯ МОДЕРАЦИЯ (ООО и расхождения)

Включается, если требуется проверка документов человеком.

**Шаг 1. Триггеры перехода в ручной режим**

- Несовпадение Ф.И.О. в аккаунте и по ИНН ИП в базе ФНС.
- Ввод ИНН юридического лица (ООО).

**Шаг 2. Системное сообщение партнеру** Система выводит модальное окно:

- *Текст: Ваши данные приняты. Для их верификации необходимо подключить сотрудника. Мы проверим информацию и пришлем вам уведомление.*
- Статус реквизитов меняется на **«На проверке»**.
- Сотруднику уходит уведомление что поступили реквизиты на верификацию.

**Шаг 3. Работа сотрудника и автоматизация после одобрения**

1. Сотрудник проверяет данные в админ-панели и нажимает «Верифицировано».
2. **Автоматика:** Система подтягивает данные из реестра (Наименование, ОГРН, Адрес) и заполняет соответствующие поля в профиле партнера.
3. Партнеру уходит системное уведомление (Email / Пуш) об успешной проверке и предложении ввести платежные реквизиты.

**Шаг 4. Завершение пути (Ввод банковских реквизитов)** При следующем заходе в «Продукты» партнер видит плашку:

- *Текст: *Юридические данные верифицированы. Пожалуйста, заполните платежные реквизиты для завершения настройки.
- Открывается форма ввода: **Наименование банка, БИК, Расчётный счёт, Корреспондентский счёт, Получатель**.
- После сохранения банковских данных переход к след.шагу - см. Раздел продукты - доступ.

## КОНТУР 3. ПЕРЕВЕРИФИКАЦИЯ ПРИ ИЗМЕНЕНИЯХ

1. При ручном изменении ИНН или Ф.И.О. в статус «Верифицировано» сбрасывается.
2. Система повторяет процесс **Контура 1** (автозаполнение и проверка через ФНС).
3. Если новая проверка не проходит автоматически — заявка снова уходит на ручную модерацию. На это время выплаты технически замораживаются.
