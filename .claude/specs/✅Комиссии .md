
**Назначение раздела:** Главный финансовый архив и накопитель данных. Сюда попадают все транзакции после того, как они были зафиксированы в разделе ручного ввода. Раздел служит для глубокой аналитики доходов компании (DS), выплат партнерам и контроля прибыли по каждой отдельной сделке.

## ЧАСТЬ 1. ИНТЕРФЕЙС И ПОЛЬЗОВАТЕЛЬСКИЙ ПУТЬ (UI)

Раздел представляет собой многоуровневую таблицу: есть основные строки (данные по сделке) и вложенные списки (цепочка наставников).

### 1.1. Блок фильтрации

- **ФИО Партнера:** Поиск сделок конкретного консультанта.
- **Партнер в цепочке:** Уникальный фильтр. Позволяет найти все сделки, с которых партнер получал пассивный доход (ГП), находясь в середине структуры, даже если саму продажу делал не он.
- **ФИО Клиента, № контракта:** Стандартный поиск.
- **Дата (От и До):** Фильтр по периодам (вызов календаря).
- **Поставщик:** фильтр по поставщику
- **Комментарии:** поиск по словам в комментариях к транзакциям
- **Кнопка Очистить фильтры.**

### 1.2. Основная таблица (Реестр транзакций)

Отображает верхнеуровневую экономику самой сделки. Есть возможност скрыть/показать продукт и програму. Также скрыть или показать доп.настройки - своя коммисия 


**Колонки:**

- **Индикатор периода (Цветной квадратик):**
  - Синий — открытый (текущий) период.
  - Серый — закрытый (замороженный) период.
- **№, Открыт, Клиент, Дата, Комментарий, Параметр, Транзакция:** Базовые данные контракта и платежа.
- **% DS:** Комиссия платформы.
- **Доход DS:** Доход компании в абсолютных цифрах.
- **Доход DS (USD / RUB):** Конвертация в нужные валюты.
- **Без НДС RUB / Без НДС USD:** Чистая выручка платформы, из которой будут высчитываться выплаты партнерам.

### 1.3. Вложенная таблица (Детализация цепочки выплат)

При клике на строку транзакции она разворачивается (аккордеон), показывая всю маркетинговую цепочку людей, получивших деньги с этой сделки.

**Колонки детализации:**

- **Партнёр:** Ф.И.О. наставника (от прямого продавца вверх по структуре).
- **ЛП / ГП:** Начисленные баллы.
- **Квалификация:** % согласно уровня квалификации 
- **Баллы:** баллы партнера и цепочки за транзакцию
- **Комиссия:** Сумма заработка партнера в рублях.
- **Прибыль ДС:** Чистый профит компании с конкретной транзакции.

## ЧАСТЬ 2. БИЗНЕС-ЛОГИКА (БЭКЕНД)

Раздел работает по строгим правилам финансового учета. Главный принцип, озвученный на видео: **«То, что закрыто, не должно подвергаться никаким правкам»**.

**1. Жесткая заморозка периодов (Серые квадраты)** Как только отчетный месяц закрыт, все транзакции этого периода навсегда становятся серыми.

- В них **невозможно** изменить ни сумму, ни валюту, ни цепочку партнеров ни удалить.
- Если обнаруживается историческая ошибка (недоплатили / переплатили), она компенсируется исключительно созданием новой корректирующей записи в текущем месяце через раздел «Прочие начисления». База прошлых периодов неприкосновенна.

**2. Управление открытыми периодами (Синие квадраты)** Пока месяц не закрыт (квадратик синий), система допускает два действия:

- **Пересчет цепочки (Клик по синему квадрату):** Если клиент написал заявление и перешел к другому партнеру, администратор нажимает на синий квадрат. Система заново строит цепочку наставников и пересчитывает их комиссии с учетом новой структуры. (это можно убрать если система сразу будет подтягивать изменения и проводить пересчеты) 
- **Удаление/редактировать:** Дать возможность редактировать дату, сумму или валюту напрямую в ячейке. Возможност удалить транзакцию. 

**3. Формула контроля «Прибыли компании»** Система автоматически считает чистую прибыль компании с каждой транзакции по формуле: Прибыль ДС = (Доход DS без НДС) минус (Сумма всех начисленных комиссий партнерам в цепочке). Это позволяет администраторам быстро проверять экономику: если математика бьется, значит алгоритмы распределения денег по структуре сработали корректно.
