**Бизнес-смысл:** Мастер-отчёт по специфике самих контрактов. Используется для анализа типов договоров, сроков их действия и графиков выплат (авансы, рассрочки).

**Тип отчёта:** Детализированный реестр договоров.

| **Название поля** | **Тип данных** | **Пример значения** | **Описание логики** |
|----|----|----|----|
| **Номер / Поставщик** | Строка | ```javascript
1213ПЛ
```, ```javascript
Кэпитал Борд
``` | ID сделки и провайдер. |
| **Дата транзакции** | Timestamp | ```javascript
2026-02-01
``` | Дата проведения в системе. |
| **Сумма в RUB** | Число | ```javascript
1648800.00
``` | Общая сумма сделки. |
| **Комиссия / Прибыль** | Число | ```javascript
17665.71
``` / ```javascript
29442.85
``` | Распределение денег (Партнер / Компания). |
| **Тип выплаты** | Строка | ```javascript
standard
```, ```javascript
upfront
```, ```javascript
year1
``` | Схема вознаграждения (аванс, за 1-й год, стандартная). |
| **Кол-во оплат (факт)** | Целое число | ```javascript
5
``` или ```javascript
1
``` | Сколько фактических платежей совершил клиент. |
| **Срок контракта** | Целое число | ```javascript
10
``` или ```javascript
1
``` | Длительность договора в годах/месяцах (может быть пустым для разовых сделок). |
| **Дата открытия** | Timestamp | ```javascript
2026-02-26
``` | Фактическая дата начала действия договора. |
