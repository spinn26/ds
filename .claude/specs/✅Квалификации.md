
**Назначение раздела:** Просмотр статистики по квалификациям, статусам и объемам продаж (ЛП, ГП, НГП) всех партнеров. Данные выводятся в виде сравнительной таблицы за два смежных месяца.

## ЧАСТЬ 1. ИНТЕРФЕЙС И ФИЛЬТРАЦИЯ (UI)

### 1.1. Панель управления (Верхний блок)

Служит для поиска нужных партнеров и настройки вывода данных.

- **Отчётный месяц (Календарь):** Выбор месяца и года (например, Март 2026), за который нужно показать данные.
- **ФИО Партнера:** Текстовое поле для поиска конкретного человека.
- **Квалификация:** Выпадающий список уровней (от 1 [Start] до максимального).
- **Активность:** Выпадающий список (Активный / Неактивный / Терминирован).
- **Чекбокс «Показать только ненулевые логи»:** Если галочка стоит, таблица скрывает «пустых» партнеров (тех, у кого личные и групповые объемы равны нулю и в текущем, и в прошлом месяце).
- **Кнопки:** Обновить данные (для ручной подгрузки свежей информации -  можно убрать если не надо) и Очистить фильтры.
- **Пагинация:** Строка внизу таблицы (например, «Стр. 1 из 27 Всего записей: 786»).

   ![](uploads/cd33e2eb-7dc8-4165-94d2-e6618f249c11/a7817262-d102-41d0-8995-1969fc2bdf3c/image.png "aspect=1")

### 1.2. Основная таблица (Сравнение периодов)

Таблица показывает динамику партнера: слева его общие данные, затем его результаты за предыдущий месяц, и правее — результаты за выбранный месяц. Сортировка списка по умолчанию идет по алфавиту (ФИО).

**Блок 1: Общая информация**

- **Партнер:** Ф.И.О.
- **Активность:** Текущее состояние (Активный / Неактивный / Терминирован).

**Блок 2: [Предыдущий месяц]** *(Например, Февраль 2026)*

- **Квалификация:** Единый уровень партнера в прошлом месяце.
- **ЛП:** Личный объем (число).
- **ГП:** Групповой объем (число).
- **ОП:** План/факт (число)
- **НГП:** Накопительный групповой объем (число).

**Блок 3: [Текущий месяц]** *(Например, Март 2026)*

- **Квалификация:** Единый уровень партнера в текущем месяце.
- **ЛП:** Личный объем.
- **ГП:** Групповой объем.
- **ОП:** План/факт (число)
- **НГП:** Накопительный групповой объем.
- **Действие (Иконка «Часы»):** Кнопка для просмотра полной истории за все месяцы.

   ![](uploads/cd33e2eb-7dc8-4165-94d2-e6618f249c11/dd9ac301-d5bb-45d0-9cf2-7afe1dcb9800/image.png "aspect=1")

### 1.3. Боковая панель «История квалификаций»

Всплывающее окно, которое открывается при клике на иконку «Часы» в строке любого партнера. Показывает всю его историю на платформе.

- **Заголовок:** «История квалификаций [Ф.И.О.]».
- **Сортировка:** Новые месяцы всегда отображаются сверху (по убыванию).
- **Колонки:** Месяц, ЛП, ГП, НГП, Квалификация.

## ЧАСТЬ 2. БИЗНЕС-ЛОГИКА РАЗДЕЛА

Раздел является исключительно информационным (Read-Only). Никакие данные здесь вручную не редактируются, они только собираются из других разделов (транзакции, начисления).

**Системные правила вывода данных:**

1. **Скрытые профили:** Система по умолчанию не выводит в этот список удаленные аккаунты и тестовые профили (аккаунты разработчиков).
2. **Единая квалификация:** У партнера есть только один показатель статуса за месяц. Система больше не сравнивает "номинальную" и "расчетную" квалификацию — выводится единый фактический уровень (например, 4 [FC]).
3. **Динамическое сравнение:** При выборе любого месяца в календаре, система всегда автоматически подтягивает данные за предшествующий ему месяц, чтобы сотрудник мог сравнить рост или падение объемов и статуса.
