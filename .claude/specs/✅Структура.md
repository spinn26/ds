
Страница представляет собой инструмент для просмотра и управления MLM-структурой партнеров.

## 1. Права доступа и видимость (Разграничение ролей)

Система строго разделяет объем отображаемых данных в зависимости от роли пользователя. Доступные фильтры при этом работают только в рамках видимой зоны:

- **Для роли «Сотрудник»:** Видит **всю структуру сети целиком**. Дерево строится с самого верхнего уровня и показывает всю глубину всех существующих веток.
- **Для роли «Партнёр»:** Видит **исключительно свою структуру** (себя и всех партнёров, которые находятся под ним: своих приглашённых, их приглашённых и так далее в глубину). Партнёр не видит своих наставников, ветки, находящиеся выше него, и параллельные структуры.

## 2. Логика интерфейса таблицы

Данные визуализируются по принципу интерактивного дерева (вложенных списков).

- **Разворот структуры:** У каждого партнёра, под которым есть команда, отображается видимая кнопка/иконка для разворота списка людей следующего уровня.
- **Визуальная иерархия:** При раскрытии списка каждый следующий уровень (вниз по структуре) обязательно имеет визуальный отступ вправо для наглядности.
- **Объединение сущностей:** Исторические данные по «ФК» и «Резидентам» в интерфейсе объединяются и выводятся в одной общей колонке **«Партнёры»**.

## 3. Фильтрация данных

Пользователям доступна расширенная фильтрация для работы с таблицей.

- **Ф.И.О. партнёра:** Три раздельных поля поиска (отдельно по Фамилии, Имени, Отчеству).
- **Дата рождения:** Поиск по конкретной дате или выбор диапазона (например, по году, месяцу).
- **Квалификация:** Выпадающий список с возможностью множественного выбора квалификаций.
- **Уровни:** Выпадающий список с возможностью множественного выбора уровней. 
- **Статус:** Множественный выбор из закрепленного списка (Зарегистрирован, Активен, Терминирован, Исключен).
- **Личные продажи (ЛП):** Ввод диапазона значений («от» и «до»).
- **Групповые продажи (ГП) и Накопленные ГП (НГП):** Ввод диапазона значений («от» и «до»).
- **Терминация:** Поиск по диапазону, конкретной дате, месяцу или году.
- **Город:** Текстовый поиск или выбор из списка.

## 4. Логика отображения колонок таблицы

Данные по партнеру в таблице загружаются и рассчитываются в пределах **текущего календарного месяца** (с 1 числа текущего по 1 число следующего месяца).

- **Партнёр:** Выводятся Фамилия, Имя и Отчество (с момента регистрации).
- **Уровень:** Абсолютный номер уровня человека в общей структуре (основатель = 0, его первая линия = 1 и т.д.).
- **Квалификация:** Текущая квалификация партнёра по маркетинг-плану.
- **Статус:** Текущий системный статус партнёра.
- **Дата смены статуса:** Расчетное поле.
  - Если статус «Активен» отображается дата активации 12 месяцев (см. Статусную схему)
  - Если статус “Зарегистрирован” отображается дата до которой надо активироватся (90 дней с момента регистрации) - см. Статусную схему.
  - + вывести Накопленый ЛП - от даты ативации (партнер чтобы оставатся активным должен сделать 500 балов личными продажами сумарно за год)
- **ЛП:** Сумма личных продаж партнёра (за месяц). 
- **ГП:** Сумма групповых продаж всей ветки под этим партнёром (за месяц).
- **НГП:** Сумма накопленных групповых продаж за весь период нахождения в сети.
- **Количество клиентов:** Подсчёт общего числа клиентов, закрепленных за партнёром.
- **Количество контрактов:** Подсчёт общего числа оформленных контрактов.
- **Количество партнёров:** Суммарное количество всех людей, приглашённых данным партнёром в сеть (сумма статусов «Резидент» и «ФК» - старая модель).

## 5. Бизнес-логика пересчета MLM-дерева

Система автоматически поддерживает актуальность связей «Наставник — Приглашенный» на основе следующих алгоритмов:

- **Добавление нового партнёра:** При регистрации новичок закрепляется за своим прямым пригласителем с помощью реф.ссылки. Система автоматически определяет его уровень (Уровень пригласителя + 1). Данные о новичке моментально прокидываются вверх по цепочке — он добавляется в структуру видимости не только прямого наставника, но и всех вышестоящих лидеров ветки вплоть до ТОПа.
- **Удаление (Терминация/Исключение) партнёра:** При исключении из структуры система фиксирует дату удаления. После этого запускается алгоритм очистки: система проходит вверх по цепочке наставников этого человека и удаляет его из их списков активной команды.
- **Изменение структуры (Перенос партнёров):** Если партнёр или целая ветка меняет наставника, система запускает глобальный пересчёт. Сначала стираются старые системные связи, затем сверху вниз проставляются новые актуальные уровни, а после — снизу вверх заново собираются списки команд. Это гарантирует, что дерево всегда отображается корректно и без разрывов. ВАЖНО -  при этом по начислениям перестройка проходит только уже по новым -  по закрытым периодам смены в структуре не должны влиять на начисления коммисий парнерам.
