**Назначение раздела:** Визуальный мониторинг текущих статусов партнеров, отслеживание дедлайнов (плановых дат смены статуса) и ручное управление жизненным циклом профилей (смена статусов и дат задним числом).

## ЧАСТЬ 1. БЛОК УПРАВЛЕНИЯ И ФИЛЬТРЫ (UI)

Верхняя панель страницы предназначена для поиска партнеров по их Ф.И.О., текущему состоянию или временным отрезкам. Фильтры адаптированы под актуальную статусную схему (4 статуса).

**Поля фильтрации:**

- **ФИО партнера (Текстовое поле):** Поиск по полному или частичному совпадению.
- **Статус (Выпадающий список):** Выбор из актуальных значений: Зарегистрирован, Активен, Терминирован, Исключен.
- **Дата регистрации с [ДД.ММ.ГГГГ] по [ДД.ММ.ГГГГ] (Календари):** Фильтр по дате заведения аккаунта.
- **Дата активности с [...] по [...] (Календари):** Фильтр по дате перехода партнера в активный статус.
- **Плановая дата терминации с [...] по [...] (Календари):** Фильтр по вычисляемому дедлайну (когда система планирует отключить партнера).
- **Дата терминации с [...] по [...] (Календари):** Фильтр по фактической дате закрытия профиля.
- **Кнопка:** Очистить фильтры.
- **Пагинация:** Строка вида Стр. 1 из 62 Всего записей: 1844 с кнопками перелистывания.

## ЧАСТЬ 2. ОСНОВНАЯ ТАБЛИЦА (РЕЕСТР)

Таблица выводит таймлайн (историю дат) по каждому партнеру в виде линейного списка.

**Колонки таблицы:**

1. **Партнёр:** Ф.И.О. пользователя.
2. **Статус:** Текущее состояние профиля (выводится в виде цветной плашки, например: зеленая для «Активен», красная для «Терминирован»).
3. **Зарегистрирован:** Точная дата присвоения первого статуса при регистрации на платформе.
4. **Активен:** Фактическая дата выполнения условий и перехода в статус (если условия еще не выполнены — поле пустое).
5. **Будет терминирован (Дедлайн):** Плановая дата, рассчитанная системой, до которой партнеру необходимо выполнить условия активности (выделяется предупреждающим цветом, например, желтым).
6. **Терминирован:** Фактическая дата закрытия профиля и отвязки структуры.
7. **Сумма ЛП от даты активации:** показывает сколько партнер уже сумарно сделал баллов от даты активации (каждый год обнуляется)
8. **Действие (Крайняя правая колонка):** Иконка «Карандаш» (Редактировать) для ручного управления строкой.

## ЧАСТЬ 3. ЛОГИКА СТРАНИЦЫ И РУЧНОЕ УПРАВЛЕНИЕ (MANUAL OVERRIDE)

Раздел позволяет сотрудникам не только просматривать данные, но и принудительно переопределять системные даты и статусы в обход автоматики (например, для корректировок из-за задержек выплат или индивидуальных решений руководства).

**Механика ручного редактирования:**

1. При клике на иконку «Карандаш» в строке партнера открывается модальное окно **«Управление статусом»**.
2. В окне доступны для изменения следующие поля:
   - **Новый статус:** Выпадающий список (Зарегистрирован / Активен / Терминирован / Исключен).
   - **Фактическая дата:** Календарь (Date picker). Позволяет выбрать любую дату, **включая даты задним числом** (ретроспективно) или будущим числом.
   - **Комментарий (Обязательное поле):** Текстовое поле для обоснования ручного вмешательства (например, «*Перенос даты терминации на 1-е число по согласованию с фин. директором*»).
3. **Кнопка:** Сохранить изменения.

**Системные правила при ручном редактировании:**

- Если сотрудник меняет дату в таблице на прошлую (например, с 15.04 на 01.04), интерфейс страницы должен немедленно обновиться и отображать новую дату в соответствующей колонке.
- База данных фиксирует эту новую введенную дату как приоритетную для любых дальнейших вычислений.
- **Аудит-лог (Скрытая логика):** Любое использование кнопки «Сохранить изменения» в этом окне обязательно логируется на бэкенде (ID сотрудника, дата и время изменения, старое значение, новое значение, текст комментария) для защиты от несанкционированных манипуляций.
