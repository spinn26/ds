**Назначение раздела:** Единый центр обучения и онбординга пользователей. Раздел содержит подробные руководства по работе с каждым модулем платформы, снижая нагрузку на службу технической поддержки (ТП).

## ЧАСТЬ 1. РОЛЕВАЯ МОДЕЛЬ (РАЗДЕЛЕНИЕ ДОСТУПОВ)

Раздел аппаратно разделен на две независимые базы знаний, чтобы пользователи видели только тот контент, который относится к их уровню доступа.

### 1.1. Инструкции для Партнёров

- **Где отображается:** В личном кабинете партнера (фронтенд платформы).
- **Контент:** Инструкции по регистрации, прохождению верификации (ввод ИНН/реквизитов), акцепту документов, сдаче тестов, открытию продуктов (в т.ч. InSmart), работе с реферальными ссылками и выводу средств. FAQ по часто задаваемым вопросам.

### 1.2. Инструкции для Сотрудников (Бэк-офис)

- **Где отображается:** В панели администратора.
- **Контент:** Инструкции по закрытию периодов, добавлению новых продуктов, модерации реквизитов, обработке заявлений на смену наставника, работе с реестрами транзакций, отчетами и историей перестановок.

## ЧАСТЬ 2. СТРУКТУРА И НАВИГАЦИЯ РАЗДЕЛА

### 2.1. Каталог инструкций (Главная страница раздела)

- **Группировка по разделам платформы:** Инструкции визуально разбиты на папки/категории, названия которых дублируют пункты главного меню платформы (например: *«Менеджер контрактов», «Раздел Продукты», «Финансы и Выплаты», «Статусная схема»*).
- **Поиск:** Текстовая строка сквозного поиска по названиям и тексту инструкций.

### 2.2. Навигация внутри статьи (Быстрый переход)

Каждая инструкция, независимо от объема, должна содержать интерактивное оглавление.

- **План подразделов (Table of Contents):** Блок в самом начале статьи, содержащий кликабельный список всех подзаголовков.
- **Якорные ссылки (Anchor links):** При клике на пункт плана экран должен плавно (smooth scroll) прокручиваться к соответствующему текстовому или видео-блоку.
- **Кнопка «Наверх»:** Плавающая кнопка в правом нижнем углу экрана для быстрого возврата к оглавлению.

## ЧАСТЬ 3. ФОРМАТ КОНТЕНТА (КАРТОЧКА ИНСТРУКЦИИ)

Каждая инструкция оформляется по единому стандарту и обязательно содержит два формата подачи информации, чтобы закрыть потребности разных типов восприятия (визуалов и читателей).

### 3.1. Видео-формат

- **Встроенный плеер:** Каждому подразделу или всей инструкции целиком сопутствует видеоролик с демонстрацией экрана.
- **Загрузка:** Видео подтягивается либо по прямой ссылке (YouTube/Vimeo/внутренний сервер), либо загружается через админку.

### 3.2. Текстовый формат (Лонгрид)

Подробная текстовая расшифровка процесса с скриншотами, дополняющая видео.

- **Пошаговый алгоритм:** Текст разбивается на нумерованные списки (Шаг 1, Шаг 2...).
- **Скриншоты:** Возможность вставки изображений интерфейса с выделением нужных кнопок.
- **Информационные плашки (Callouts):** Выделенные цветом блоки для акцентирования внимания:
  - *Внимание (Красный):* Для критических системных предупреждений (например, о невозможности редактировать закрытый период).
  - *Заметка (Желтый/Синий):* Для полезных советов и примечаний.

## ЧАСТЬ 4. УПРАВЛЕНИЕ КОНТЕНТОМ (ДЛЯ АДМИНИСТРАТОРОВ)

Администраторы с соответствующими правами должны иметь удобный инструмент для создания и обновления инструкций.

- **Редактор (WYSIWYG):** Визуальный текстовый редактор для форматирования текста, вставки таблиц, скриншотов и видео без знания HTML.
- **Автогенерация оглавления:** Система должна уметь автоматически собирать «План подразделов» на основе стилей заголовков (H2, H3), которые администратор использует в тексте.
- **Статусы публикации:** Возможность сохранить инструкцию в «Черновик» перед тем, как опубликовать её для партнеров или сотрудников.
