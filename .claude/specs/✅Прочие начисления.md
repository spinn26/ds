**Назначение раздела:** Ручное управление балансами партнеров. Позволяет сотруднику точечно добавлять или списывать рубли и баллы ЛП и в НГП с указанием комментария и даты операции.

## ЧАСТЬ 1. ИНТЕРФЕЙС И ПОЛЬЗОВАТЕЛЬСКИЙ ПУТЬ (UI/UX)

Экран разделен на три основных блока: фильтры, рабочая таблица (реестр) и всплывающее окно добавления новой операции.

### Блок 1. Фильтры и поиск

Предназначен для быстрого поиска конкретных операций в истории начислений.

- **Ф.И.О. консультанта:** Текстовое поле (поиск по базе партнеров).
- **Комментарий:** Текстовый поиск по содержимому примечаний к операциям.
- **Тип:** Выпадающий список (Рубли / Баллы).
- **Дата (от и до):** Поля с календарем для выбора периода.
- **Кнопка «Очистить фильтры».**

### Блок 2. Реестр начислений (Рабочая таблица)

Здесь отображается история всех ручных операций. *В идеале таблица должна иметь кнопку **«Экспорт»** (или «Скачать»), чтобы выгружать отфильтрованные данные по дате и партнеру в CSV/Excel для бухгалтерии.*

**Колонки таблицы:**

- **Дата:** Дата, указанная при создании операции.
- **Партнёр:** Ф.И.О. партнера, с балансом которого произведена операция.
- **Сумма:** Числовое значение со знаком «+»  - начисление (зеленым цветом) или «-»  - удержание (красным цветом) и указанием типа (Руб / Баллы).
- **Комментарий:** Текстовое примечание (например, "Билет на съезд", "Премия за конкурс").
- **Действие (Иконка корзины):** Возможность удалить ошибочно заведенное начисление или удержание.
- **Действие (иконка карандаша):** Возможность отредактировать заведенные данные

### Блок 3. Добавление нового начисления (Модальное окно)

Открывается при нажатии на кнопку **«Добавить начисление»** над таблицей.

**Поля для заполнения -  все обязательные:**

1. **Тип операции (Радио-кнопка):**
   - Рубли (Влияет на финансовый баланс партнера для выплат).
   - Баллы (Влияет на квалификационные счетчики объемов ЛП и НГП -  *ВАЖНО при начислении баллов начисляются только баллы - в деньги они не конвертируются при том*).
2. **Партнёр (Поле с поиском):** Выбор консультанта из общего списка (с автодополнением).
3. **Сумма (Числовое поле):** Ввод суммы. Для списания используется знак минуса (например, -33000). Для начисления — просто число.
4. **Дата (Календарь):** Важное поле. Позволяет выбрать дату, чтобы операция была учтена в правильном расчетном периоде (например, выбрать последний день прошлого месяца, чтобы сумма удержалась/начислилась задним числом).
5. **Комментарий (Текстовое поле):** Обязательное поле для указания причины операции (например, "За билет на съезд").

## ЧАСТЬ 2. БИЗНЕС-ЛОГИКА (БЭКЕНД)

При нажатии кнопки «Сохранить начисление» в модальном окне, система выполняет следующие действия:

**1. Проверка валидности:** Система проверяет, заполнены ли все обязательные поля (Партнер, Сумма, Дата, Комментарий) и корректен ли формат суммы.

**2. Запись в реестр:** Операция добавляется в таблицу «Прочие начисления» и становится доступной для поиска и выгрузки.

**3. Обновление балансов партнера (Маршрутизация по типу):** Система определяет тип операции и вносит изменения в профиль партнера:

- **Если выбрано Рубли:**
  - Сумма (с учетом знака + или -) прибавляется/вычитается из финансового баланса партнера (того, что отображается в разделе «Комиссии» и доступен к выплате).
  - *Важно:* Эта операция НЕ влияет на квалификационные баллы (ЛП, НГП).
- **Если выбрано Баллы:**
  - Сумма (с учетом знака) прибавляется/вычитается из объема продаж партнера (ЛП, НГП).
  - *Важно:* Начисление/списание баллов таким образом должно корректно отражаться в маркетинговой структуре (например, если баллы начислили, они должны учитываться для поддержания статуса, но не должны генерировать финансовую комиссию для вышестоящих наставников, как это происходит при обычной продаже).

**4. Удаление операции (Функция отмены):** Если сотрудник нажимает иконку корзины напротив операции в реестре:

- Запись удаляется из таблицы.
- Система выполняет **обратную (реверсивную) транзакцию** на балансе партнера (если было списано 100 баллов, они возвращаются; если было начислено 500 рублей, они списываются).
